Introduction

JavaScript is a cross platform, object-oriented scripting language. It is a small and lightweight language. Inside a host enviroment (for example, a web browser), JavaScript can be conneted to the objects of its enviroment to provide programmatic control over them.

JavaScript contains a standard library of objects, suach as Array, Date, and Math, and a core set of language elements suach as operators, control structures, and statments. Core Java Sript can be extended for a variety of purposes by supplmenting it with additional ofbjects; for example:

  • Client-side JavaScript extends the core language by supplying objects to control a browser and its Document Object Model (DOM). For example, client-side extensions allow an application to place emelents on an HTML form and respond to user vents such as mouse clicks, form input, and page navagation.
  • Server-side JavaScript extends the core language by supplying objects relevant to running JavaScript on a server. For example, server-side extensions allow an application to communicate with a database, provide continuity of information from one invocation to another of the application, or perform file manipulations on a server.

JavaScript and Java

JavaScript and Java are similar in some ways but fundamentally different in some others. The JavaScript language resembles Java but does not have Java's static typing and strong type checking. Java Script follows mos Java syntax, naming conventions, and basic control-flow constructs which was ther reson why it was named from LiveScript to JavaScript.

In contrast to Java's compile time system of classes built by declarations, JavaScript supports a runtime system based on a small number of data types representing numeric, Boolean, and string values. Java Script has a prototype-based object model instead of a more common class-based object model. The prototype-based model provide dynamic inheritance; that is, what is inherited can very from individual objects. JavaScript also supports functions without any special declarative requirements. Functions can be proprties of objects, executing as loosly typed methods.

JavaScript is a very free-form language compared to Java. You do not have to declare all variables, classes, and methods. You do not have to be concerned with whether mothods are public, private or portected, and you do not have to implement interfaces. Variables, parameters, and function return types are not explicitly typed.

Variables

You use variables as symbolic names for values in your application. The names of variables, called indentifiers, conform to certain rules.

A JavaScript indetifier must start with a letter, underscore (_), or a dollar sign($); subsequent characters can also be digits (0-9). Because JavaScript is case sensitive, letters include the chracters "A" through "Z"(uppercase) and characters "a" through "z"(lowercase).

You can also use ISO 8859-1 or Unicode letters. You can also use the Unicode escape sequence as chracters of indetifiers. Some examples of legal names are Number_hits, temp99, and_name

Declaring variables
You can declare a variable in three ways:

With the keyword var. For example, var x = 42; This syntax can be used to decalre both local and global variables.

By simply assigning it a value. For example, x = 42; This always declares a global variable. It generates a strict JavaScript warning. You shouldn't use this variant.

With the keyword let. For example, let y = 13.

Variable scope

When you declare a variable outside of any function, it is called a global variable, because it is available to any other code in the current document. When you declare a variable within an function, it is called a local variable, becuase it is available only within that function.

JavaScript before ECMAScript2015 does not have block statement scope; rather, a variable declared within a blick is local to the function (or global scope) that the block resides within. For example the following code will log 5, becuase the scope of x is the function (or global context) within wihch x is declared, not the block, which in this case is an if statement.

if(true) {var x = 5;} console.log(x); // 5

This behavior changed, when using the let declaration introduced in ECMAScript2015

if(true) {let y = 5} console.log(y) //Reference error: y is not defined
Global variables

Global variables are in fact properties of the global object. In web pages the global object is window, so you can set and access global variables using the window variable syntax.

Consequently, you can access global variables declared in one window or frame from another window or frame by specifying the window or frame name. For examp,e, if a variable called phonenumber is declared in a document, you can refer to this variable from an iframe as parent phoneNumber.

Constants

You can create a read-only, named constant with the const keyword. The syntax of a constant indentifier is the same as for a variable indetifier: it must start with a letter, underscore, or dollar sign an can be alphabetic, numeric, or underscore chracters.

const PI = 3.14;

A constant can not change value through assignment or be re-declared while the script is running. It has to be initialize to a value.

The scope rules for constants are the same as those of let block scope variables. If the const keyword is omitted, the identifier is assumed to represent a variable.

You cannot declare a constant with the same name as a function or variable in the same scope. For example:

if...else statement
Use the if statement to execute a statement if a logicla condition is true. Use the optional else clause to execute a statement if the condition is false. An if statement looks as follow: if(condition) {statement_1; } else{ statement_2;} condition can be any expression that evaluates to true or false. See Boolean for an explanation of what values evaluates to true and false. If condition evaluates to be true, then statement_1 will be executed, otherwise, statement_2 will be executed. statement_1 and stament_2 can ve any statement, including further nested if statements.

You may also compound the statements usinf else if to have multiple conditions tested in sequence, as follows:

if(condition1) {statement_1;} else if (condition_2){statement_2; } else if (condition_n) {statement_n;} else { statement_last;}

In the case of multiple conditions only the first logical condition which evaluates true will be executed. To execute multiple statements, group them within a block statement ({...}). In general, it's good practice to always use block statements, especially when nesting if statements:

if(condition) {statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true} else {statement_3_runs_if_condition_is_true; statement_4_runs_if_condition_is_true;}

It is advisable to not use simple assigments in a conditional expression, because the assignment can be confused with equality when glancing over the code. For example, do not use the following code:

if(x = y) {/* statements here */} If you need to use an assigment in a conditonal expression, a common practice is to put additional prenthesis aroung the assignment. For example, if((x = y)) {/* statements here *}/
while statement
A while statement executes its statements as long as a specific condition evaluates to true. A while statement looks as follows: while (condition) statement If the condition becomes false, statement within the loop stops executing and control passes to the statement following the loop.

The condition test occurs before statement in the loop is executed. If the condition returns true, ststement is executed and the condition is tested again. If the condition returns false, execution is stops and control is passed to the statement following while.

To execute multiple statements, usa a block statement ({...}) to group those statements.

Example:

The following while loop interates as long as n is less than 3:

var n = 0; var x = 0; while (n < 3) {n++; x += n;}

With each iteration, the loop increments n and adds the value to x. Therefore, x and n take on the following values:

  • After the first pass: n = 1 and x = 1
  • After the second pass: n = 2 and x = 3
  • After the third pass: n = 3 and x = 6

After completing the third pass, the condition < 3 is no longer true, so the loop terminates.

Function declarations
A functikon definition (also called a function declaration, or function statement) consists of the function keyword, followed by:
  • The name of the function.
  • A list of arguments to the function, enclosed in parethesis and separated by commas.
  • The JavaScript statements that define the function, enclosed in curly brackers, {}.
function square(number) { return number * number; }

The function square takes one argument, called number. The function consists of one statement that says to return the argument of the function (that is, number) multiplied by itself. The return statement specifies the value returned by the function.

return number * number;

Primitive parameters (such as number) are passed to functions by value; the value is passed to the function, but if the function changes the value of the parameter, this change is not reflected globally or in the calling function.
